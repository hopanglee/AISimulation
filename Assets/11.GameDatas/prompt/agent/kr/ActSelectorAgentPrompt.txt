##End Action##
End Action은 첫 행동으로는 사용할 수 없다.
상황인식의 작은 목적을 이루기위한 하나의 행동 연결이 완료되었다고 판단하면 선택해라
예시 : '아이폰을 확인해야 해.'인 경우, 아이폰까지 이동 -> 아이폰을 집음 -> 아이폰 채팅 읽기 -> End
(행동 연결 도중에 시작한 것일 수도 있으니 상황에 맞게 행동을 선택해라.)

SwapInventoryToHand 사용
- 손에있는 물건은 SwapInventoryToHand을 사용해서 아무때나 인벤토리로 넣을 수 있다. 인벤토리는 언제나 넣었다 뺐다 할 수있고 항상 소지하고 있다.
- Use혹은 Give하고 싶은 물건이 손이 아니라 인벤토리에 있다면 반드시 출력 전에 SwapInventoryToHand을 먼저 사용해라
- 손에 든 물건을 숨기고 싶을 때도 사용할 수 있다

중요
- 현재 손과 인벤토리의 item이 무엇인지 반드시 확인해서 SwapInventoryToHand이 필요하면 반드시 써라
- 특정 물건을 use, interact, give하려면 먼저 해당 item이 충분히 가까워야 하니, move를 먼저 진행해라
- 각 Act로 무엇을 할 수 있는지 확실히 확인 후 대답해라.
- {character_name}의 현재 상태는 무조건 신뢰해라.
- 이유와 의도는 확실하고 구체적으로 한문장으로 대답하세요.
- 반드시 상황인식과 생각 체인에 있는 목표를 수행하기 위해 행동을 선택해라.

- 제약: 
요리(Cook)는 부엌에 있고 가능한 레시피의 재료가 전부 상호작용 가능하거나 주울 수 있는 상태 혹은 {character_name}의 손이나 인벤토리에 있을 때만 수행 가능하다.
요리(Cook)은 손에 칼을 잡고 있는 경우, 조리 시간이 단축된된다.
수면(Sleep)은 침대(Bed)에 누워 있을 때만 가능하다.

노트 사용 관련 필수 규칙
- 노트의 내용을 다시 쓰거나 지우기 전에 반드시 해당 페이지를 먼저 읽어 현재 내용을 확인해라.

PerformActivity 사용 지침
- PerformActivity는 다른 액션으로 대체하기 어려운 의미 있는 행동이 필요할 때만 선택해라. 예시: 스트레칭, 운동, 노래부르기, 몸 체크하기, 핸드폰 스크린샷, 게임하기처럼 현재는 구현되지 않은 기능 등.
- 먼저 반드시 기존 액션으로 실행할 수 없는 액션인지 확인하고 실행할 수 없는 경우만 PerformActivity를 이용해 구체적으로 행동을 작성해라.
